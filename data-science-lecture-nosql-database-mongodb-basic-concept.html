MongoDB Basic Concept

드디어 MongoDB에 대해서 알아볼 차례입니다. 기본적인 개념부터 알아보도록 하겠습니다.

MongoDB는 10gen사에서 C++언어를 이용해 개발한 NoSQL 솔루션 입니다. 상당히 범용적으로 사용이 가능하다는 특징이 있으며 당연히 스키마를 고정하지 않고 사용합니다. (schema-less)

데이터를 구조화해서 JSON형태로 저장하는 방식을 이용하고 있습니다. 또한 JOIN이 불가능하기 때문에 JOIN이 필요없도록 데이터를 설계해야 합니다.

한가지 기억해야 할 점이 있는데 MongoDB는 메모리맵 형태의 파일엔진 DB이기 때문에 메모리에 상당히 의존적 입니다. 즉, 메모리의 크기가 성능을 좌우하게 되며 가용 메모리를 넘어설 경우 성능이 급격히 저하되는 문제점을 가지고 있습니다.

일반적으로 사용하는 데이터는 쌓아놓고 삭제가 없는 경우가 적합합니다. 대표적인 예로 로그데이터같은 것이 있을 수 있습니다.

당연히 RDBMS의 강점인 Transaction처리가 필수인 금융, 결제, 빌링과 같은 부분에는 부적합합니다.

Document Data Model

MongoDB는 Document Data Model을 사용합니다. 간단하게 특징을 살펴보면,

  • 속성의 이름과 값으로 이루어진 쌍의 집합으로 구성
  • 속성은 문자열이나 숫자, 날짜뿐 아니라 다양한 형식이 올 수 있다.
  • 배열 혹은 다른 Document를 지정하는 것도 가능
  • 하나의 Document에 필요한 모든 정보를 담아야 합니다.
  • JOIN이 불가능하기 때문에 하나의 쿼리로 모두 해결이 되게끔 설계해야 합니다.

아래의 그림은 간단한 Document형태의 구조입니다.

JSON

JSON은 JavaScript Object Notation의 약자입니다. MongoDB에서 사용하는 Document는 이 JSON 형식으로 되어 있습니다.

JSON은 다음과 같은 특성을 가집니다.

  • lightweight 데이터 교환방식
  • 사람이 읽고 쓰기 쉽고, 기계가 파싱하고 생성하기 쉬움
  • XML이 가지는 유연성과 구조적 데이터 표현이 가능하면서 XML의 오버헤드를 줄일 수 있음
  • 따라서 송수신 데이터 형식으로 사용하기 적합하며 클라이언트 측에서도 처리 효율이 좋음

MongoDB 구성

MongoDB는 수평적 확장이라는 특징을 가집니다. 쉽게 말하면 한 대 이상의 서버로 구성하는 것이 일반적이라는 말입니다. MongoDB는 메모리 사용 가능량에 대비해서 성능이 좌우되기 때문에 독립된 서버에서 실행하는 것이 좋습니다.

기본적으로 MongoDB는 mongod라는 실행파일을 단위로 실행합니다.

데이터를 한 곳에만 저장하면 데이터 손상 시 문제가 발생하기 때문에 적어도 보통 3개 이상의 Replica단위로 구성해 데이터를 복제합니다.(샤딩)

또한 mongos라는 라우터를 이용하여 마치 한대의 데이터베이스 서버처럼 사용이 가능합니다.

간단하게 그림으로 표현하면 다음과 같습니다.

MongoDB를 사용하는 사용자의 90% 정도는 Linux 머신을 이용합니다. 그 이유는 많은 수의 machine이 필요하고 상대적으로 OS 비용이 적은 Linux를 선호하기 때문입니다. 또한 MongoDB 자체의 성능이 메모리에 많이 좌우되기 때문에 메모리 관리에 특히 뛰어난 Linux 또는 Unix 베이스의 운영체제를 선호합니다. Windows는 설치하고 사용하기 편하죠.

MongoDB가 지원하는 언어는 다음과 같습니다.

  • Java / JavaScript
  • Perl / PHP / Python
  • Ruby / Scala

MongoDB 다운로드 및 설치

이제 MongoDB를 실제로 설치해 보죠.

MongoDB Official Home으로 접속해서 Product > Community Edition > Community Server를 선택하면 다운로드 받을 수 있습니다.

현재 버전이 5.0.5입니다. Windows용 msi 파일을 다운로드 한 후 default 형태로 설치합니다. Compass가 같이 설치되는데 우리는 일단 console에서 간단하게 알아볼 것입니다.

우리는 설치 시 MongoDB를 Windows Service로 등록해서 사용한다고 했으니 MongoDB라는 이름으로 서비스가 실행되고 있을 것입니다. 확인 한번 하고 가죠.

PATH를 등록해야 사용하기 편합니다. C:\Program Files\MongoDB\Server\5.0\bin 경로를 환경변수 PATH에 등록한 후 shell을 실행시켜 보겠습니다.

서비스로 실행하는 mongod는 특정 mongod.cfg파일을 이용해 환경설정을 하고 시작됩니다.

C:\Program Files\MongoDB\Server\5.0\bin\mongod.cfg 파일을 보면 아래와 같이 되어 있는데 여기서 storage > dbPath라는 부분을 보면 특정 경로가 잡혀 있습니다. 이 경로가 default로 사용되는 Database 경로입니다.


# mongod.conf

# for documentation of all options, see:
#   http://docs.mongodb.org/manual/reference/configuration-options/

# Where and how to store data.
storage:
  dbPath: C:\Program Files\MongoDB\Server\5.0\data
  journal:
    enabled: true
#  engine:
#  wiredTiger:

자 이제 shell을 실행시켜 보겠습니다.

cmd를 이용해 도스창을 열고 mongo 명령어를 입력하면 아래와 같이 shell이 실행됩니다.

만약 다른 서버에 접속하려면 mongo ip:port 형식으로 실행하면 다른 컴퓨터의 MongoDB에 접속이 가능합니다.

이제 명령어를 하나씩 살펴보겠습니다.

가장 먼저 MongoDB를 안전하게 종료하는 방법입니다. 우리는 지금 Windows Service로 등록해서 사용하고 있긴 하지만 해당 서비스를 명령어를 이용해서 안전하게 종료하는 방법을 알아보겠습니다.


> use admin;

> db.shutdownServer();

위와 같은 명령어를 이용하면 MongoDB 자체를 안전하게 종료할 수 있습니다. 서비스를 살펴보면 다음과 같이 MongoDB가 중지되어 있는 것을 확인할 수 있습니다.

Windows Service에서 다시 MongoDB를 기동시킨 후 shell로 다시 접속합니다.

MongoDB 데이터베이스 생성 및 조회

MongDB안에 만들어져 있는 database 목록을 조회하려면 다음과 같이 합니다. 참고로 이 작업은 shell을 이용해도 좋지만 다른 MongoDB client 프로그램을 이용해도 좋습니다. 사실 사용하기에는 shell은 불편한 점이 많습니다. 개인적으로 JetBrain의 DataGrip을 선호하는데 유료버전(30일 trial)입니다. 교육용 license를 받을 수 있는 경우 고려해 볼 만 합니다.

MongoDB collection 생성 및 조회

MongoDB는 RDBMS의 Table에 해당하는 collection이라는 개념을 가지고 있습니다. MongoDB에서 저장하는 단위라고 보시면 됩니다. 기존의 RDBMS는 Schema를 정해놓고 그 Schema에 맞는 데이터만 저장할 수 있었는데 collection은 어떤 종류의 데이터라도 다 저장할 수 있습니다.

collection을 생성할때는 db.createCollection(name, [options]) 명령어를 사용합니다.

name은 생성하려는 컬렉션의 이름이며 option은 document 타입으로 구성된 해당 컬렉션의 설정값입니다. options 매개변수는 선택적인(Optional) 매개변수로서 생략하셔도 되고, 필요에따라 사용하면 됩니다.

사용할 수 있는 option을 살펴보면 다음과 같습니다.

  • capped : Boolean, 이 값을 true 로 설정하면 capped collection 을 활성화 시킵니다. Capped collection 이란, 고정된 크기(fixed size) 를 가진 컬렉션으로서, size 가 초과되면 가장 오래된 데이터를 덮어씁니다. 이 값을 true로 설정하면 size 값을 꼭 설정해야합니다. 또 주의해야 할 점은 capped collection은 삭제가 되지 않습니다. 대신 insert 속도가 빠릅니다.

  • autoIndexId: Boolean, 이 값을 true로 설정하면, _id 필드에 index를 자동으로 생성합니다. 기본값은 false 입니다.

  • size : number, Capped collection 을 위해 해당 컬렉션의 최대 사이즈(maximum size)를 bytes로 지정합니다.

  • max: number, 해당 컬렉션에 추가 할 수 있는 최대 갯수를 설정합니다.


db.createCollection("books")

db.createCollection("articles", {
    capped: true,
    size: 6142800,
    max: 10000
})

db.people.insertOne({"name": "velopert"})

show collections;

MongoDB collection 삭제

collection을 제거 할 땐 drop()을 사용합니다. 이 명령어를 사용하기 전에 사용 할 데이터베이스를 우선 설정해야합니다.


use test;

db.people.drop();

MongoDB 데이터 저장

이번에는 collection에 데이터를 저장하는 방법에 대해서 알아보겠습니다. insertOne(), insertMany() 와 같은 함수를 사용하여 document를 추가 할 수 있습니다.

이 명령어를 사용하기 전 데이터를 추가 할 데이터베이스를 당연히 선택해주어야합니다. 또한 배열형식의 인자를 전달해주면 여러 다큐먼트를 동시에 추가 할 수 있습니다. 또한 collection의 document 리스트를 확인할때는 db.COLLECTION_NAME.find() 함수를 사용합니다.

아래의 코드를 보시죠.


db.books.insertOne({
    "name": "MongoDB Guide",
    "author": "홍길동"
});

db.books.insertMany([
    {"name": "Book1", "author": "아이유"},
    {"name": "Book2", "author": "김연아"}
]);

// 아래와 같은 경우 우리가 원하는 방식으로 입력이 되지 않습니다.
// insertOne, insertMany를 구분해서 사용해야 합니다.

db.books.insertOne([
    {"name": "Book1", "author": "아이유"},
    {"name": "Book2", "author": "김연아"}
]);

db.books.find();

당연히 JavaScript 코드와 함께 사용해서 document를 입력할 수 있습니다.


for (i=0; i<10; i++) {
    db.books.insertOne(
        {"name": "Book" + i, "no": 'No.' + i}
    );
}

db.books.find();

MongoDB 데이터 삭제

collection에 저장된 document를 삭제할 수 있습니다. removeOne(), removeMany() 함수를 사용하여 document를 제거 할 수 있습니다.


use test;

// collection books 삭제
db.books.drop();

// collection books 생성
db.createCollection("books")

// 아래처럼 capped collection을 만들면
// 삭제가 되지 않습니다. 대신 빠른 insert 처리가 가능합니다.
//db.createCollection("books", {
//    capped: true,
//    size: 6142800,
//    max: 10000
//})

// collection안의 데이터(document) 조회
db.books.find();

// books collection에 데이터 입력
db.books.insertMany([
    {"name": "Book1", "author": "아이유"},
    {"name": "Book2", "author": "김연아"}
]);

for (i=0; i<5; i++) {
    db.books.insertOne(
        {"name": "Book" + i, "no": 'No.' + i}
    );
}

// 모든 데이터 조회
db.books.find()

// 데이터 조회
db.books.find({"name": "Book1"})

// 원하는 데이터 삭제(1개만 삭제)
db.books.deleteOne({"name": "Book1"})
db.books.find({"name": "Book1"})

// 원하는 데이터 삭제(여러개 삭제)
db.books.deleteMany({"name": "Book1"})
db.books.find({"name": "Book1"})

// 모든 데이터 삭제
db.books.deleteMany({})
db.books.find()

MongoDB 데이터 조회

이번에는 collection안의 document를 조회하는 방법에 대해서 자세히 알아보겠습니다.

document를 조회하는 함수는 find() 입니다. 다음과 같이 사용합니다.

db.COLLECTION_NAME.find(query, projection)

여기에 사용되는 매개변수는 다음과 같습니다.

  • query: document, Optional(선택적). 다큐먼트를 조회할 때 기준을 정합니다. 기준이 없이 컬렉션에 있는 모든 다큐먼트를 조회 할때는 이 매개변수를 비우거나 비어있는 다큐먼트 {} 를 사용합니다.

  • projection: document, Optional. 다큐먼트를 조회할 때 보여질 field를 설정합니다.

반환값(return 값)은 cursor 입니다. criteria에 해당하는 document들을 선택하여 cursor를 반환합니다. cursor는 query 요청의 결과값을 가르키는 pointer로 cursor객체를 통하여 보이는 데이터의 수를 제한 할 수 있고, 데이터를 sort 할 수 도 있습니다. 이는 10분동안 사용되지 않으면 만료된다는 것도 추가적으로 기억해 둬야 합니다.

자 그럼 간단한 데이터를 입력하고 기본적인 조회부터 해 보기로 하죠.


// 데이터베이스를 선택합니다.
use test;

// 만약 articles collection이 존재한다면 삭제합니다.
db.articles.drop();

// articles collection을 생성합니다.
db.createCollection("articles");

// document를 여러개 입력합니다.
db.articles.insertMany([
    {
        "title" : "article01",
        "content" : "content01",
        "writer" : "홍길동",
        "likes" : 0,
        "comments" : [ ]
    },
    {
        "title" : "article02",
        "content" : "content02",
        "writer" : "신사임당",
        "likes" : 23,
        "comments" : [
            {
                "name" : "이선희",
                "message" : "좋은글"
            }
        ]
    },
    {
        "title" : "article03",
        "content" : "content03",
        "writer" : "강감찬",
        "likes" : 40,
        "comments" : [
            {
                "name" : "",
                "message" : "좋은글"
            },
            {
                "name" : "장범준",
                "message" : "좋은글"
            }
        ]
    }
]);

// 모든 document 조회
db.articles.find();

// writer 값이 '신사임당'인 Document 조회
db.articles.find({ 'writer': '신사임당' } );      // 1건 조회

// likes 값이 30 이하인 Document 조회
db.articles.find({ 'likes': { $lte: 30 } } );   // 2건 조회

맨 마지막에 연산자가 등장했습니다. 이 연산자는 잘 알아둘 필요가 있습니다. 여기에서는 자주 사용되는 몇가지 연산자를 위주로 알아보겠습니다.

  • 비교(Comparison) 연산자

간단한 예를 들어보기로 하죠.


// likes 값이 10보다 크고 30보다 작은 Document 조회
db.articles.find({ 'likes': { $gt: 10, $lt: 30 } } );   // 1건 조회( likes : 23 )

// writer 값이 배열 ['이순신', '을지문덕', '강감찬', '홍길동'] 안에 속하는 값인 Document 조회
db.articles.find({ 'writer': { $in: ['이순신', '을지문덕', '강감찬', '홍길동'] } } );
// 2건 조회( writer : 홍길동, 강감찬 )

// comment 값이 '장범준'인 Document 조회
db.articles.find({ 'comments.name': '장범준' } );  // 1건 조회
  • 논리(Logical) 연산자

일반적인 논리 연산을 수행하는 연산자 입니다.

간단한 예를 들어보기로 하죠.


// title 값이 'article01' 이거나, writer 값이 '신사임당' 인 Document 조회
db.articles.find({ $or: [{'title': 'article01'}, {'writer': '신사임당'}] });  // 2건 조회

// writer 값이 '홍길동' 이고 likes 값이 10 미만인 Document 조회
db.articles.find({ $and: [{'writer': '홍길동'}, {'likes': {$lt: 10}}] });  // 1건 조회

// 위의 구문은 다음과 같이 작성해도 됩니다.
db.articles.find({ 'writer': '홍길동', likes: { $lt: 10} });  // 1건 조회
  • 정규식(Regular Expression) 연산자

$regex 연산자를 통하여 document를 정규식을 통해 찾을 수 있습니다. 단, 정규식에 대한 기본 지식이 있어야 사용이 가능합니다.


// title 값이 'article0'로 시작하고 끝이 1 또는 2로 끝나는 Document 조회
db.articles.find( { 'title' : { $regex: /article0[1-2]/ } } )   // 2건 조회
  • $where 연산자

$where 연산자를 통하여 javascript expression을 사용 할 수 있습니다. 아래의 예를 보시죠.


// comments field 가 비어있는 Document 조회
db.articles.find( { $where: 'this.comments.length == 0' } )   // 1건 조회
  • $elemMatch 연산자

$elemMatch 연산자는 Embedded Documents 배열을 쿼리할때 사용됩니다. 우리 예에서는 comments 가 Embedded Document에 속합니다.


// comments 중 '장범준'이 작성한 글이 있는 Document 조회
db.articles.find( { 'comments': { $elemMatch: { 'name': '장범준' } } } )
  • projection

End.
