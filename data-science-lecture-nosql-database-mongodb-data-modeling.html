NoSQL Data Modeling

이번에는 NoSQL의 데이터 모델링 개념과 특징을 알아보고 주요 NoSQL 데이터 모델링 기법에 대해서 살펴보겠습니다.

Denomalization

관계형 데이터베이스와는 다르게 NoSQL은 비정규화를 지향합니다. 비정규화를 지향한다는 의미는 데이터의 중복을 허용한다는 의미와 같은 말입니다. RDBMS는 정규화를 진행하고 데이터의 중복을 허용하지 않는게 기본 원칙인데 반해 NoSQL은 쿼리 프로세싱을 간단하게 하거나 최적화 하기 위해 같은 데이터를 여러 Document나 테이블에 복제하는 것을 허용합니다.

비정규화로 얻는 이점과 단점은 명확합니다. 쿼리당 IO 횟수나 쿼리 데이터 사이즈는 분명히 줄어들고 쿼리 수행의 복잡도 역시 낮아지지만 전체 데이터 사이즈는 당연히 커지게 됩니다.

하지만 빅데이터 솔루션 특성상 데이터 사이즈가 커지는 오버헤드는 충분히 해결할 여지가 있습니다.

Aggregates

유연한 스키마 속성이라고 할 수 있습니다. 이 유연한 스키마 덕분에 복잡하고 다양한 구조의 내부 요소를 가지고 있는 데이터 클래스를 구성할 수 있습니다.

또한 1:n과 같은 관계를 최소화하여 결과적으로 JOIN과 같은 연산을 줄일 수 있고 추후 확장성과 변동성에 대응이 가능한 복잡하고 다양한 비지니스적 요소를 담을 수 있습니다.

대부분은 NoSQL 솔루션은 기본적으로 유연한 스키마를 지원합니다.

간단하게 예를 들어보죠. NoSQL의 schema-less 특성을 이용하면, 데이터 모델을 하나의 테이블로 합칠 수 있습니다. 심지어 지금 고려중인 아닌 파일 포맷도 추후에 얼마든지 지원가능합니다.

Application Side Join

대부분의 NoSQL 솔루션이 JOIN 연산을 지원하지 않습니다. 대용량 데이터에 대한 빠른 응답 성능과 확장성, 가용성을 최우선 목적으로 하기 때문에 쿼리 타임 조인을 최대한 피하여 데이터 모델을 구성하는 것을 가정합니다.

NoSQL 솔루션은 중복을 허용하기 때문에 해당 데이터의 Update시에 많은 비용이 발생할 수 있습니다. 이런 데이터인 경우 그 부분은 차라리 RDBMS로 관리하는 것이 효과적입니다.

그리고 아래 그림처럼 NoSQL 내에서 JOIN이 되지 않기 때문에 Application단에서 JOIN을 수행하는 식으로 처리해야 합니다.

NoSQL Data Modeling 기법

일반적인 Modeling 기법에는 다음과 같은 것들이 있습니다. 우리는 이 중 두가지만 알아보도록 하겠습니다.

  • Atomic Aggregates
  • Enumerable Keys
  • Dimensionality Reduction
  • Index Table
  • Composite Key Index
  • Aggregation with Composite Keys
  • Inverted Search - Direct Aggregation

Composite Key

Composite Key는 하나 이상의 필드를 deliminator(구분자)를 이용하여 구분지어 사용하는 방법을 의미합니다. 즉, 문자열로 구성된 하나의 키를 특정 구분자를 이용해 구분지어서 사용한다는 것이죠. Ordered Key/Value Store같은 경우 이를 이용하여 order by나 grouping 기능을 구현할 수 있습니다. 또한 Key를 기준으로 여러개의 서버에 나누어서 전체 서버에 부하가 골고루 분산 될 수 있도록 데이터를 분산시킬 수 도 있습니다.

Inverted Search Index

value의 내용을 key로 하고, key의 내용을 반대로 value로 하는 패턴을 의미합니다. 가장 흔하게 사용되는 곳이 바로 검색엔진입니다.

검색엔진은 사이트의 모든 페이지를 검색 로봇이 검색해서 문서내에 단어들을 색인하여 URL에 맵핑해서 저장해 놓습니다.

검색은 단어를 key로 검색하기 때문에 value에 검색 키워드가 들어가 있을 경우 효과적인 검색이 불가능합니다.

따라서 검색 키워드를 키로 하고 URL을 value로 하는 테이블을 다시 만든다음 검색 키워드로 검색을 하면 신속하게 URL을 찾아낼 수 있을 것입니다.

Tree Aggregation

Tree Aggregation은 Tree 구조 자체를 하나의 Value에 저장하는 방식을 의미합니다. JSON이나 XML을 이용해 Tree 구조를 정의하고 Value에 저장하는 방식이고 Tree자체가 크지 않고 변경이 많이 없는 경우에 적합한 방식입니다.
